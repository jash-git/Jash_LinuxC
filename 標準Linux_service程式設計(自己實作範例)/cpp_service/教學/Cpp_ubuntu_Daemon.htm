C++基础:ubuntu下Daemon如何自动启动

  • 时间：2011-09-16　来源：嗨考网　责任编辑：
　　前段时间在写了个Daemon(守护历程）,但不知若何开机自动启动，在看到debian系统的初始化后茅塞顿开，网上介绍Daemon的文章不行偻指算，嗨考网就不介绍了, Daemon在各个Unix平台下的事理是一样的，只是完成细节上的差别而已，此刻写了一个精简的Daemon,具体的可以参照《Unix环境高级编程》
　　以ubuntu为例（ubuntu为debian的分支，所以系统编程和设置装备摆设极为相似），
　　代码如下
　　//mydaemon.c
　　#include<unistd.h>
　　#include<stdlib.h>
　　#include<sys/param.h>
　　#include<sys/types.h>
　　#include<sys/stat.h>
　　void init_daemon(void)
　　{
　　int pid;
　　int i;
　　if(pid=fork())
　　exit(0);
　　else if(pid<0)
　　exit(1);
　　//是第一子历程
　　setsid();//第一子历程成为新的会话组长和历程组长, 并与节制终端分手
　　if(pid=fork())
　　exit(0);//竣事第一子历程
　　else if(pid< 0)
　　exit(1);
　　//第二子历程不再是会话组长
　　for(i=0;i< NOFILE;++i)//封锁打开的文件描述符
　　close(i);
　　chdir("/");//改变工作目录到/
　　umask(0);//重设文件建树掩模
　　return;
　　}
　　int main()
　　{
　　init_daemon();//使历程成为Daemon
　　//以下可以添加功能代码
　　while(1)
　　{
　　sleep(10);//睡眠一分钟
　　}
　　return 0;
　　}
　　输入呼吁
　　gcc mydaemon.c -o mydaemon
　　./mydaemon
　　如不美观成功，可以用ps -efjc查看，我们会发现这个历程的父历程为init,到这里守护历程成功运行，
　　良多Daemon如ftpd, httpd都设为开机自动运行，下面谈判若何经由过程添加剧本让系统来启动Daemon，按照系统初始化事理一文（本blog已转载）介绍，
　　我在/etc/init.d/中添加一个新建一个剧本，为mydaemon, 剧本如下，假设我的daemon目录为/home/eyesmart/Daemon
　　#! /bin/sh
　　# /etc/init.d/mydaemon: start the my daemon.
　　PATH=/bin:/usr/bin:/sbin:/usr/sbin
　　binpath=/home/eyesmart/Daemon/mydaemon
　　test -f $binpath || exit 0
　　. /lib/lsb/init-functions
　　case "$1" in
　　start)
　　log_begin_msg "Starting mydaemon..."
　　start-stop-daemon --start --quiet--exec $binpath
　　log_end_msg $?
　　;;
　　stop)
　　log_begin_msg "Stopping mydaemon..."
　　start-stop-daemon --stop --retry TERM/1/TERM/1/TERM/4/KILL --quiet --oknodo--exec $binpath
　　log_end_msg $?
　　;;
　　restart|force-reload)
　　$0 stop
　　sleep 1
　　$0 start
　　;;
　　*)
　　log_success_msg "Usage: $binpath {start|stop|restart|force-reload}"
　　exit 1
　　esac
　　exit 0
　　大致是这样，剧本可以按需要写的再详尽些,做到这里按说已经完成，可是unix有个run level这个特征，所以我们必需对于特定的run level建树一个链接，
　　一般linux是运行level 2，那么我们输入下列呼吁ln -s /etc/init.d/mydaemon /etc/rc2.d/S50mydaemon.
　　到这老迈功乐成，当你重启系统的时分你可以在开悔悟程中看到自己些的轨范所显示的信息。
　　其他Unix系统环境在细节上会有所差别，但事理一样。
