Ubuntu Apache的安装和配置以及CGI的设置

分类： 嵌入式 2012-06-08 17:22 169人阅读 评论(0) 收藏 举报

2011-12-01 14:40:26|  分类： Linux  |  标签：linux   ubuntu   apache    |字号大中小 订阅

在学习Python CGI中，需要在Ubuntu下安装Apache，所以今天在Ubuntu11.04上对Apache的安装和配置进行了学习。
一.Apache的安装
       在网上查了些资料有两种安装方式：一种是从Apache网站上下载tar文件到自己的电脑上安装；另一种是直接通过网络安装。为了方便我就使用了第二种方式。
       Ubuntu提供了强大的apt-get install命令，在终端输入：sudo apt-get install apache2   
       这样就能方便快捷的安装Apache。但是，这里有个不便的地方，就是你不知道Apache被安装到了什么位置。

二.启动、停止Apache服务
       Apache的启动和停止文件是：/etc/init.d/apache2
       启动命令：sudo apache2ctl -k start
       停止命令：sudo apache2ctl -k stop
       重新启动命令：sudo apache2ctl -k restart(sudo /etc/init.d/apache2 restart)
       重新启动这里有个问题，当我重启服务的时候会报错：“Could not reliably determine the server’s fully qualified domain name, using 127.0.1.1 for ServerName”。我在网上搜了一下说是httpd.conf中ServerName IP：Port和Listen IP：Port不一致，即端口不一致。所以在httpd.conf文件中增加“ServerName localhost:80”即可。
       说明下httpd.conf是一个Apache的配置文件，在/etc/apache2目录下，在后面会有介绍。
       在修改httpd.conf文件之前还需要修改该文件的权限，否则是没有权限修改的，命令为：sudo chmod 777 /etc/apache2/httpd.conf
       顺便提下查看Apache的版本，使用的命令是：apache2ctl -v 结果会显示：
      Server version: Apache/2.2.17 (Ubuntu)
      Server built:   Nov  3 2011 02:13:18


三.Apache的配置文件
       普通Apache发行版本的配置文件是：/etc/apache2/httpd.conf，但发现这个文件是空的，其实这个是用户自己来进行配置的，后面会说到。
       严格地说，Ubuntu的Apache配置文件是/etc/apache2/apache2.conf，Apache在启动的时候会自动读取这个文件的配 置信息，而其他的一些配置文件，如httpd.conf等，则是通过Include指令包含进来的。可以在apache2.conf中找到这些 Include行：
       ● 动态模块的配置
         Include mods-enabled/*.load
         Include mods-enabled/*.conf
       ● 用户自己的配置
         Include httpd.conf
       ● 端口监听的配置
         Include ports.conf
       ● 一般性的配置语句片断
         Include conf.d/
       ● 虚拟主机的配置指令
         Include sites-enabled/
      我们完全可以把所有的设置放在apache2.conf或者httpd.conf或者任何一个配置文件中，这种划分是一种比较好的习惯。

      1.Web文档根目录
       安装完Apache后最重要的一件事情就是要知道Web文档的根目录在什么地方。对于Ubuntu而言，默认的是/var/www。那我们是怎么知道的呢？apache2.conf里并没有描述，httpd.conf也是空的，那肯定是在其他的配置文件中。后来搜索发现是在/etc/apache2/sites-enabled/000-default中，里面有这样的内容：
       DocumentRoot /var/www
          <Directory />
              Options FollowSymLinks +ExecCGI
              AllowOverride None
          </Directory>
         <Directory /var/www/>
              Options Indexes FollowSymLinks MultiViews
              AllowOverride None
              Order allow,deny
              allow from all
         </Directory>
     这里的DocumentRoot中把根目录设置为了/var/www
   
     2.sites-enabled和sites-available目录
     刚才在apache2.conf中发现了sites-enabled目录，而在/etc/apache2下还有一个sites-available目录。这两个目录放的是什么的？其实，sites-available里面放的才是真正的配置文件，而sites-enabled目录存放的只是一些指向这里的文件的符号连接，通过 ls -l 命令查看到/etc/apache2/sites-enabled中的000-default是指向/etc/apache2/sites-available/default的。这里存放的是虚拟主机的配置指令。
     虽然sites-available是存放有效的内容，但不起作用，只有用 ln 命令链接到enabled过去才能起作用。
     
     3.mods-enabled和mods-available目录
     这两个目录的性质和上面说到sites-enabled、sites-available一致，这里存放apachegon功能模块的配置文件和链接。

     4.ports.conf设置
     这里设置了Apache使用的端口。如果需要调整默认的端口设置，建议编辑这个文件。还有一种修改方式，可以把apache2.conf中的Include prots.conf一行去掉，在httpd.conf中设置自己想用的端口。

四.CGI的配置
      1.首先在根目录下建一个cgi-bin的目录，即/var/www/cgi-bin/。cgi程序都放在这个cgi-bin的目录中，不能放在其他的目录中。
     
      2.打开/etc/apache2/sites-enabled/000-default，找到以下的内容：

         ScriptAlias /cgi-bin/ /usr/lib/cgi-bin/
         <Directory "/usr/lib/cgi-bin/">
                      AllowOverride None
                     Options +ExecCGI -MultiViews +SymLinksIfOwnerMatch
                     Order allow,deny
                     Allow from all
         </Directory>

         修改为：

         ScriptAlias /cgi-bin/ /var/www/cgi-bin/
         <Directory "/var/www/cgi-bin">
                      AllowOverride None
                     Options +ExecCGI -MultiViews +SymLinksIfOwnerMatch
                     Order allow,deny
                     Allow from all
                     AddHandler cgi-script cgi
         </Directory>

         其中的/usr/lib/cgi-bin/修改为/var/www/cgi-bin/，并添加了AddHandler cgi-script cgi。如果Options没有ExecCGI这项，记住一定要加上。

      3.重启Apache服务：sudo /etc/init.d/apache2 restart

      4.如果还不能访问cgi脚本，则再修改/etc/apache2/mods-enabled/cigd.load文件。在原本的LoadModule cgid_module /usr/lib/apache2/modules/mod_cgid.so下，再加上：

         AddHandler cgi-script .cgi .pl .py .sh这句，再重启Apache服务。

      5.修改cgi程序的权限。CGI程序属性一定要设为可运行（755），而与CGI有关的HTML文件的目录如果要被CGI程序写入，其权限一定要设为可写（666）。其命令为：sudo chmod 755 ~/simple.cgi

      至此，通过浏览器输入相应的url应该能顺利的访问到cgi程序了，如http://localhost/cgi-bin/simple.cgi。如果还不能的话，估计就是cgi程序本身的问题了，可以通过查找Log，发现其中的错误。Apache的log放置在/var/log/apache2下。
